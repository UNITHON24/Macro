# 음성 키오스크 Macro

[English](README.md)

UNITHON 2024에서 만든 접근성 프로토타입입니다. 사용자의 음성 주문을 구조화된 메뉴 항목으로
받아 고정된 키오스크 화면을 대신 조작합니다. 운영 중인 서비스가 아니라 해커톤 산출물이며,
통합 경계와 안전장치, 한계를 검토할 수 있는 형태로 보존합니다.

## 보여주는 것

- 16 kHz 마이크 스트리밍과 WebSocket 기반 VAD 처리
- 로컬 주문 허브를 통한 구조화 주문의 HTTP 전달
- OCR을 이용한 메뉴 카테고리·페이지·화면 좌표 초기 인덱싱
- 검증된 메뉴 항목을 실제 키오스크 조작 순서로 바꾸는 결정론적 내비게이션
- 기본 dry-run, 주문 상한, 운영자 확인을 거치는 수동 결제

음성 인식 백엔드와 키오스크 웹 애플리케이션은 팀의 외부 구성 요소이며 이 저장소에 포함되지
않습니다.

## 저장소 구조

| 경로 | 역할 | 상태 |
| --- | --- | --- |
| `macro_pkg/` | launcher, 초기 설정, 음성 클라이언트, 주문 허브, 키오스크 통합 | 기준 실행 경로 |
| `tests/` | 설정·메뉴 인덱스·주문 안전 경계의 외부 의존성 없는 테스트 | CI 실행 |
| `kioskMacro/` | 출처 보존을 위한 팀의 이전 클라이언트 스냅샷 | 레거시 참고 |
| `settingPack/`, 루트 유틸리티 | 초기 OCR 캡처와 실험 | 레거시 참고 |

두 클라이언트 트리를 무리하게 합치지 않은 이유는 [아키텍처 문서](docs/ARCHITECTURE.md)와
[ADR-0001](docs/adr/0001-preserve-hackathon-snapshot.md)에 기록했습니다.

## 안전한 핵심 로직 검증

아래 검증은 마우스와 마이크를 사용하거나 서버에 연결하지 않습니다.

```bash
python -m compileall -q macro_pkg tests
python -m unittest discover -s tests -v
```

GitHub Actions도 Python 3.11에서 같은 검증을 실행합니다.

## 로컬 실행

Windows 키오스크 환경과 Python 3.10 이상을 기준으로 합니다. 오디오, OCR, 화면 자동화 패키지는
운영체제별 네이티브 설정이 필요할 수 있습니다.

```powershell
py -m venv .venv
.venv\Scripts\activate
py -m pip install --upgrade pip
py -m pip install -r requirements.txt
```

전체 launcher는 이 저장소 밖의 팀 백엔드를 필요로 합니다. `KIOSK_BACKEND_DIR`에 백엔드
경로를 지정한 뒤 실행합니다.

```powershell
py macro_pkg\launcher.py
```

백엔드가 이미 8080 포트에서 실행 중이면 다음 명령을 사용합니다.

```powershell
py macro_pkg\launcherNonback.py
```

두 launcher는 기본적으로 저장소에 포함된 보정 데이터를 사용하며 OCR 보정을 실행하지 않습니다.
실제 서비스가 아닌 테스트 키오스크에서 좌표와 메뉴 인덱스를 다시 만들 때만 명시적으로 켭니다.

```powershell
set KIOSK_RUN_CALIBRATION=1
py macro_pkg\launcherNonback.py
```

보정 과정은 주문 클라이언트가 dry-run이어도 키오스크를 열고 실제 포인터를 움직입니다. 보정
중에도 PyAutoGUI 화면 모서리 failsafe를 사용할 수 있게 유지합니다.

## 안전장치

기본값에서는 포인터를 실제로 움직이지 않습니다. 실제 클릭은 명시적으로 켭니다.

```powershell
set KIOSK_DRY_RUN=0
```

클라이언트는 첫 클릭 전에 주문 전체를 검증하고, 기본적으로 메뉴 종류 10개와 메뉴별 수량 10개로
제한하며, 한 번에 주문 하나만 실행합니다. HTTP 주문 허브만 매크로 실행 경로로 사용하고
WebSocket `macro.trigger` 이벤트는 정보로만 받아 두 경로가 같은 주문을 중복 실행하지 않게 했습니다.

결제 동작은 dry-run에서만 시뮬레이션할 수 있습니다.

```powershell
set KIOSK_ALLOW_CHECKOUT=1
```

실제 클릭 모드에서는 검증된 메뉴를 장바구니에 담은 뒤 멈춥니다. 운영자가 장바구니, 수량,
금액과 현재 화면을 직접 확인하고 결제를 이어가야 합니다. PyAutoGUI 화면 모서리 failsafe가
활성화되면 남은 주문 전체를 중단합니다. 전체 설정값은
[아키텍처 문서](docs/ARCHITECTURE.md)에 있습니다.
긴급 중단 뒤에는 운영자가 키오스크 상태를 확인하고 클라이언트를 재시작할 때까지 새 주문을
거부합니다.

## 기여 경계

팀 해커톤 프로젝트입니다. 이 문서에서 개인 범위로 설명하는 부분은 `macro_pkg/`의 launcher,
설정, 패키징과 기존 매크로 통합입니다. 음성 백엔드, 프론트엔드와 앞선 팀 클라이언트 구현은
개인 작업으로 주장하지 않습니다.

## 한계

- 특정 해상도와 키오스크 레이아웃에 좌표가 종속됩니다.
- OCR은 초기 보정을 도울 뿐 실행 중 UI 변경을 제거하지 못합니다.
- 데스크톱 권한, 오디오 장치와 네이티브 라이브러리에 의존합니다.
- 주문 허브는 메모리 기반 단일 프로세스 전달 계층이며 영속 큐가 아닙니다.
- launcher가 외부 자식 프로세스를 시작하지만 감시하거나 재시작하지는 않습니다.
- 네이티브 실행 의존성은 선언했지만, 외부 의존성 없는 CI에서 취약점 감사까지 수행하지 않습니다.
- 공개 배포, 사용자 연구나 장기 운영 성과는 없습니다.

실제 제품이라면 좌표 자동화보다 애플리케이션 수준 접근성 API 또는 명시적인 키오스크 명령
계약이 더 안정적입니다.

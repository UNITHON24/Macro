# ADR-0002: 소스 없는 키오스크를 의미 기반 폐쇄루프로 제어한다

- 상태: accepted
- 날짜: 2026-07-15
- 대체 범위: ADR-0001의 “아카이브된 프로토타입이라 재설계하지 않는다” 결정

## 맥락

UNITHON 2024 시연 당시 화면 동작에 문제가 생겨 절대 좌표를 직접 넣어 주문 흐름을 마무리했다. 기준 클라이언트는 이후에도 카테고리, 페이지, 메뉴를 저장 좌표로 클릭했고 포인터 API에 예외가 없으면 성공으로 판단했다. 화면 배율, 레이아웃, 모달, 옵션, 장바구니 반영을 확인할 수 없었으며 소스가 없는 다른 키오스크에 적용하기 어려웠다.

제품 목표는 소유한 프론트엔드에 API를 추가하는 것이 아니다. 음성 기능이 없는 기존 Windows 키오스크에 별도 프로그램을 설치하고, 원본 소스나 DOM 없이 결제 화면까지 접근할 수 있게 하는 것이다.

## 결정

`macro_pkg/`를 기준 경로로 유지하면서 실행 핵심을 다음 구조로 바꾼다.

1. 대상 창을 제목과 window handle로 고정한다.
2. Windows UI Automation의 의미·Invoke 패턴을 우선 사용한다.
3. UIA에서 현재 목표를 찾지 못하면 EasyOCR로 대상 창의 현재 텍스트 위치를 다시 찾는다.
4. 키오스크 프로필에는 좌표 대신 별칭, 옵션 의미, 화면 상태, 허용 전이, postcondition을 우선 저장한다.
5. 모든 action은 재관찰한 안정 상태와 장바구니 delta를 확인해야 성공이다.
6. 좌표 fallback은 명시적 opt-in과 동일한 사후 검증을 요구한다.
7. 결제 방법 선택 화면을 terminal로 삼고 실제 결제 동작은 금지한다.
8. 주문은 SQLite로 보존하고 한 번에 하나만 claim한다. 검증된 live 장바구니 변경은 `awaiting_handoff`, action 결과가 불확실하면 `uncertain`으로 두고 운영자 확인 전에는 다음 주문을 받지 않는다.
9. 주문 허브는 모든 모드에서 설치별 공유 secret으로 주문 등록, claim과 결과 ACK를 인증한다. dry-run과 live가 같은 영속 큐를 사용하므로 토큰 없는 주문은 저장하지 않는다.

## 검토한 대안

### 모든 메뉴와 버튼을 YOLO로 학습

아이콘뿐인 완전 불투명 UI에서는 가치가 있지만 현재 대상은 한국어 텍스트가 중심이다. 메뉴·배치가 바뀔 때마다 데이터셋과 가중치 검증이 필요하고 모델 크기, 지연, 배포, 라이선스 비용이 커진다. UIA와 OCR가 모두 제공하지 못하는 아이콘이 실제 acceptance에서 확인될 때 별도 detector adapter로 평가한다.

### 좌표와 버튼 전이를 모두 데이터베이스에 저장

화면 전이 그래프 자체는 채택하지만 좌표를 상태의 정체성으로 사용하지 않는다. 좌표 그래프는 DPI와 레이아웃 변경에 취약하고 클릭 결과를 증명하지 못한다. 그래프에는 관찰 가능한 의미 상태와 postcondition을 저장하고 bounding box는 실행 직전에 구한다.

### 키오스크 프론트엔드 API 또는 DOM 제어

소유한 애플리케이션에서는 더 안정적이지만 제품 요구인 외부·레거시 키오스크에 사용할 수 없다. 팀 데모 프론트엔드는 acceptance fixture일 뿐 production dependency가 아니다.

### 모든 버튼을 자동 탐색하는 제로샷 agent

결제 UI를 무작위로 탐색하면 취소, 결제 확인, 개인정보 입력 같은 위험 동작이 발생할 수 있다. 새 키오스크는 read-only 진단과 운영자가 승인한 전이만 사용한다.

### claim 이후 timeout 자동 재시도

물리 장바구니 동작은 transaction rollback이 불가능하다. ACK 유실을 새 주문으로 재생하면 수량이 중복될 수 있으므로 불확실 상태를 보존하고 운영자가 실제 화면을 확인한다.

## 결과

키오스크마다 프로필과 Windows acceptance가 필요하므로 “아무 키오스크나 즉시 지원”하지 않는다. 대신 소스가 없는 기기라는 본래 제약을 유지하면서 오클릭과 중복 주문을 탐지 가능한 실패로 바꾼다. pure core는 CI에서 재현할 수 있지만 Windows UIA, OCR 모델, 물리 기기는 별도 검증 대상이다.

초기 팀 스냅샷 보존과 `macro_pkg/` 기준 경로라는 ADR-0001의 나머지 결정은 유지한다.
